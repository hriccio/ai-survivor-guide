# Repository Instructions

- After making repository changes, create a git commit before ending the session unless the user explicitly says not to commit.
- Keep this repository focused on a bounded ebook/manual: Markdown sources, PDF generation, release automation, and supporting assets.
- Do not introduce blog, saga, arc, episode, analytics, newsletter, or web-publishing concepts unless explicitly requested.
- Generated PDFs in `dist/` are build artifacts and should remain untracked.
